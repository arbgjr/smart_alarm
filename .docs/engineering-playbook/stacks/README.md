# Stacks

Stacks é um catálogo de referências técnicas adotadas ou recomendadas no momento.

Importante:
- Stack não é regra absoluta.
- Stack não substitui decisão registrada.
- Stack não é desculpa para parar de pensar.

## Para que existe

- Dar defaults para times e agentes
- Reduzir retrabalho em decisões repetidas
- Documentar escolhas preferidas e seu contexto
- Tornar exceções explícitas e comparáveis

## O que não é

- não é lista de tecnologias obrigatórias
- não é documento para debate de preferência pessoal
- não é substituto de ADR

## Como uma definição de stack se torna valida

Uma entrada em stack deve conter:

- Quando usar
- Quando não usar
- Trade offs
- Requisitos mínimos de operação
- Implicações de segurança
- E, quando aplicável, apontar para decisão registrada

## Como lidar com exceções

Quando alguém precisar sair do padrão:

1. justificar com evidência ou necessidade clara
2. isolar a exceção em fronteira explícita
3. registrar decisão
4. definir data de reavaliação

Se exceção virar rotina:
- atualizar stack
- mantendo a rastreabilidade da decisão

## Estrutura

- software
  linguagens e runtimes

- dados
  data platform e analytics

- devops
  ci cd, observabilidade e segurança
